# 채움 예시 — 가상 프로젝트 "단어숲" (템플릿을 어떻게 채우는지 감 잡기용)

> 이 파일은 **예시**다 — 복사해 쓰는 파일이 아니다. CLAUDE.md 템플릿의 플레이스홀더가 실제로 어떻게 채워지는지 보여준다.
> 가상 프로젝트: 초등 영단어 암기 PWA (React+TS / Hono+pg, 콘텐츠 대량 생산 + 학습 이벤트 수집).

# 단어숲 작업 하네스 (AI 전용 — 모든 모델이 동일 품질을 내기 위한 규약)

초등 3~6학년 영단어 암기 PWA. 숲을 가꾸며 단어를 외운다. 모노레포: client(React18+Vite+TS) / server(Hono+pg).
**"단어장 앱"으로 한정짓지 말 것** — 문장·듣기 모듈로 확장 예정.

## 0. 오케스트레이션 원칙

- 메인 = 계획·브리프·검증·통합·커밋만. 단어 콘텐츠 대량 저작은 서브에이전트 위임.
- 검증 통과 판정·교육 적합성 판단·커밋은 절대 위임하지 않는다.
- 위임 시 `docs/agent-brief-template.md` 8요소 브리프 필수. 경계 = `.claude/agents/README.md`.
- 규모 있는 작업은 `docs/plan-template.md`로 계획서 먼저. 버그는 `debug`, 새 로직은 `tdd` 스킬.

## 2. 불변식 (vitest 속성 테스트로 강제 — 위반 = 실패)

- `Math.random()` 금지 → `seedRng(seed)`만. 같은 seed = 같은 퀴즈 (재현성).
- 단어 데이터는 `content/words/*.md`가 정본, `src/data/`는 AUTO-GENERATED — 직접 수정 금지, `npm run build:words`로 재생성.
- 보기 4개 값 유일·정답 1개. 오답은 실제 혼동 기반(철자 유사·품사 혼동)만 — 무작위 단어 금지.
- 학년 어휘 수준 준수 (교육과정 기본 어휘 목록 밖 단어는 4학년 이하 노출 금지).

## 3. 검증 루프 (커밋 전 필수, 순서대로)

```
npx tsc --noEmit -p client && npx tsc --noEmit -p server
npx vitest run                        # 전체 — 기준선 이상
npm run build -w client
npx tsx scripts/check-word-dupes.ts   # 단어 id 중복 게이트
```
- **테스트 수 기준선: 1,204** (2026-07-08) — 줄어들면 뭔가 깨진 것.

## 5. 불변 결정 (사용자 명시 — 되돌리지 말 것)

- localStorage 키 'wordforest-save' 변경 금지 (저장 호환, 2026-07-01).
- 나무 성장 보상은 "하루 첫 학습"에만 — 반복 학습 보상 경로 추가 금지 (과사용 방지, 2026-07-05).

## 6. 데이터·개인정보 불변 결정

- 이벤트는 `client/src/analytics/types.ts` 화이트리스트 6종만. 자유 텍스트·닉네임 금지.
- 새 이벤트 = 클라 types + 서버 allowlist + DDL registry **3곳 동기화**.
- ANALYTICS_DB_URL 미설정 = no-op (학부모 동의 전 수집 방지) — 우회 금지.

## 7. 환경·커밋 규약

- Windows + PowerShell. 커밋 메시지 한국어, 큰따옴표 금지(here-string `@'...'@`).
- main 푸시 = Railway 자동 배포. `content/원본자료/`는 저작권 원본 — gitignore, `git add -A` 금지.

## 8. 자주 쓰는 사실

- 단어 md 형식이 바뀌면 파서(`scripts/word-parser.ts`)와 형식 테스트 둘 다 갱신 — 한쪽만 고치면 빌드는 되고 데이터가 조용히 빠진다.

## 채움 요령 (이 예시에서 볼 것)

1. **§2 불변식은 "실제로 깨져본 것"부터** — 상상으로 채우지 말고, init 시점엔 2~3개면 충분. 실패가 나면 추가된다.
2. **§3은 실행해본 명령만** — 예시의 4줄 전부 이 프로젝트에서 실제로 도는 명령이어야 한다.
3. **해당 없는 절은 삭제** — 이 예시엔 §1(반복 파이프라인)·§4(외부 협업)가 없다. 빈 절을 남기지 않는다.
4. **§5는 날짜와 함께** — "왜"는 decisions-log.md에, 여기엔 한 줄 요약만.
