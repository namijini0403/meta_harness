# {{프로젝트명}} 에이전트 팀 — 협응 규칙 (오케스트레이터용)

> 템플릿: `{{...}}`를 채우고 표의 예시 행을 프로젝트 실제 에이전트로 교체하라.
> 에이전트 정의 파일은 `_TEMPLATE.md`를 복사해 만든다.

메인 모델(오케스트레이터)이 도메인별 전문 서브에이전트에게 작업을 위임한다. CLAUDE.md §0의 확장.
**오케스트레이터(메인) 독점**: 계획·설계 결정·**검증·도메인 정확성 판단·커밋/머지/푸시·충돌 중재**. 절대 위임 금지.

## 별명 ↔ 에이전트 (호출용)

부르기 쉬운 별명을 부여한다. 사용자가 "○○관 가동해"처럼 별명으로 지시하면 오케스트레이터가 매핑한다.
(에이전트 `name` 필드는 ASCII여야 레지스트리에 잡히므로 영문 유지 — 별명은 각 정의의 `description` 맨 앞에도 박아둔다.)

| 별명 | 에이전트(`name`) | 한 마디 |
|---|---|---|
| {{예: 집필관}} | {{content-author}} | {{콘텐츠 대량 집필}} |
| {{예: 검수관}} | {{content-auditor}} | {{적대적 검증·버그 리포트 (읽기전용)}} |
| {{예: 운영관}} | {{ops-backend}} | {{운영 백엔드·인증}} |
| {{예: 수문장}} | {{security}} | {{보안·개인정보 감사 (packs/security-privacy 참조)}} |

## 소유권 경계 (병렬 충돌 방지 — 이게 핵심)

에이전트마다 **편집 가능한 파일 glob**을 명시한다. 경계 밖 파일이 필요하면 STOP + 보고.
경계가 겹치지 않는 에이전트끼리만 병렬 가동.

| 에이전트 | 소유(편집 가능) | 비고 |
|---|---|---|
| {{content-author}} | {{`src/content/**`}} | {{공유 헬퍼는 조심}} |
| {{content-auditor}} | (없음 — 읽기전용) | 버그 리포트만 |
| {{ops-backend}} | {{`server/src/**`}} | |
| {{security}} | {{`docs/security/**`}} | 코드는 **감사만**(지적 → 오너가 수정) |

## ⚠️ 공유·순차 파일 (절대 병렬 금지 — 오케스트레이터가 순차 디스패치)

여러 에이전트가 append하는 파일을 전부 열거한다. 한 번에 한 에이전트만.

- {{예: `src/registry/index.ts` — 모든 모듈이 등록하는 인덱스}}
- {{예: `src/routes.ts`}}

## 핸드오프 프로토콜

- {{예: "author ↔ auditor: auditor는 리포트만, 수정은 author. 리포트 형식 = 심각도·file:line·재현법·수정안."}}
- {{예: "이벤트 추가는 analytics 에이전트 단독 — 3곳 동기화(§CLAUDE.md 6). 다른 에이전트는 이벤트 추가 금지."}}
- {{예: "security 가로지르기: 코드 오너가 아닌 영역은 감사·지적만. 수정은 해당 오너에게 위임."}}

## 모든 서브에이전트 공통 규약 (각 정의에 내장할 것)

1. **커밋·푸시 절대 금지** (오케스트레이터 전담).
2. **소유 경계 밖 파일 편집 금지** — 필요 시 STOP + 보고.
3. **공유 순차 파일 단독 편집 금지** — 오케스트레이터에 요청.
4. **자체 검증 루프**(타입체크·관련 테스트)로 통과까지 자기수정. 단 **최종 게이트·도메인 정확성 인증은 오케스트레이터**.
5. 완료 보고: 변경 파일 목록 · 테스트 명령+출력 · 자체검토 결과 · 미해결 우려.
6. 해당 도메인 불변식(각 정의에 전문 수록) 위반 = 실패.
