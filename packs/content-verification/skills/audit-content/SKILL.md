**읽기전용.** 산출물을 적대적으로 스캔해 버그 리포트만 낸다. 수정은 저작 스킬/오너 에이전트의 몫.

> 짝 스킬: 저작 = `author-{{도메인}}`. 에이전트 귀속: {{감사 에이전트}} (소유 파일 없음 — 읽기전용).

## 스캐너 구성 (scanners/ 에 번호 순서로 축적)

감사 = 독립 스캐너의 모음. 각 스캐너는 실행 가능한 파일로 만들어 `scanners/NN-<이름>`에 번호순으로 축적한다.
새 버그 유형이 발견될 때마다 스캐너를 추가한다 — **감사는 실패 기록으로 자란다.**
동봉된 `scanners/01-vocab-subset.example.mjs`가 참조 구현(프레임워크 무관 Node 스크립트·종료코드로 판정) — CONFIG를 채우고 `.example`을 떼면 그대로 동작하며, 프로젝트에 테스트 프레임워크가 있으면 같은 로직을 테스트 파일로 옮겨도 된다.

| 순서 | 스캐너 | 잡는 것 (예시 — 프로젝트에 맞게 교체) |
|---|---|---|
| 01 | 전수 덤프 | 육안 확인용 — 사용자 노출 텍스트 전부를 파일로 |
| 02 | {{문법/표기 스캔}} | {{예: 조사·어미 오류}} |
| 03 | {{산술/정합 검산}} | {{예: 표시값 독립 재계산}} |
| 04 | {{인접 중복}} | {{예: 항목 간 유사도}} |
| 05 | {{메타 정합}} | {{예: 코드 ⊆ 통제 어휘, 등록 커버리지}} |
| 06 | {{노출 금지}} | {{예: 내부 코드·시크릿·식별정보 노출}} |

## 리포트 형식 (강제)

- 발견마다: **심각도(Critical/Important/Minor) · 위치(file:line 또는 항목 id) · 재현 방법 · 구체 수정안**.
- 수정하지 않는다 — 오너에게 위임 (병렬 충돌 방지).
- 위반 0이어도 "무엇을 몇 건 스캔했는지" 범위를 보고 (스캔 범위 누락이 최대 함정).

## 합리화 방지

| 유혹 | 규율 |
|---|---|
| "명백한 오타니까 그냥 고치자" | 읽기전용. 리포트만 — 수정 권한 분리가 감사의 신뢰 근거. |
| "샘플 몇 개만 보고 통과" | 전수 스캔. 스캐너가 안 보는 영역은 명시적으로 "미스캔"으로 보고. |
