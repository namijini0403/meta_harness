# 서브에이전트 위임 브리프 템플릿 (8요소 — 병렬 안전의 핵심)

> 오케스트레이터가 서브에이전트에 작업을 위임할 때 **이 템플릿을 복붙하고 델타만 채운다.**
> "에이전트가 agents/README.md를 읽겠지"는 실패한다 — **브리프가 유일하게 확실한 전달 채널**이므로
> 경계·불변식은 매번 브리프에 다시 적는다 (요약 금지, 전문 수록).

---

## 브리프 8요소

```
[1. 임무] {{한 문단 — 무엇을 만들/고칠 것인가, 완성 세트 기준(CLAUDE.md §1)}}

[2. 절대 금지]
- git 커밋·푸시 절대 금지 (오케스트레이터 전담)
- 아래 [3] 경계 밖 파일 편집 금지 — 필요하면 STOP 후 보고
- 공유 순차 파일({{목록}}) 편집 금지 — 오케스트레이터에 요청

[3. 파일 경계 (편집 가능한 파일 전체 목록 — 병렬 형제와 겹침 0)]
- {{편집 가능: 정확한 경로/glob}}
- {{읽기만: 참조 구현·정본 문서 경로}}

[4. 참조 구현] {{기존 코드 중 따라야 할 모범 예시 파일 경로 — "이 파일과 같은 구조로"}}

[5. 불변식 전문] {{CLAUDE.md §2 중 이 작업에 해당하는 불변식을 **전문 복붙** — 요약하면 위반한다}}

[6. 자체 검증 루프 (통과까지 자기수정 후 보고)]
- {{타입체크 명령}}
- {{자기 범위 테스트 명령}}
- {{전체 실행이 필수인 검사가 있으면 명시 — "-t 필터 금지" 등}}

[7. 완료 보고 형식]
- 변경 파일 목록 (전체 경로)
- 실행한 검증 명령 + 출력 요약
- 자체검토에서 발견·수정한 것
- 미해결 우려 / 오케스트레이터 판단 필요 사항

[8. 범위 밖 발견 처리] 타 파일의 버그·위반을 발견하면 고치지 말고 보고만 (병렬 형제와 충돌 방지)
```

## 오케스트레이터 체크리스트 (브리프 발송 전)

- [ ] 병렬로 띄우는 에이전트들의 [3] 경계가 서로 겹치지 않는가? (겹치면 순차로)
- [ ] 공유 순차 파일을 이번 작업에서 누가 만지는가? 한 번에 하나인가?
- [ ] [5]에 불변식을 **전문**으로 넣었는가? (요약·링크만 넣으면 위반한다)
- [ ] 이어받기 대비: 에이전트가 죽으면 `git status`로 산출물 확인 → "기존 산출물 검수부터" 지시로 재가동

## 산출물 수령 후 (메인 통합 게이트)

1. 보고된 변경 파일과 `git status` 대조 (보고 밖 파일 변경 = 경계 위반, 검토)
2. CLAUDE.md §3 전체 검증 루프 실행 (에이전트의 자체 검증은 참고일 뿐, 정본은 메인 실행)
3. 육안 확인 (HARNESS.md §6-C) — 사용자가 실제 보는 산출물 직접 읽기
4. 통과 시에만 커밋
