# 불변 결정 로그 (decisions-log.md 로 복사해 사용)

> **사용자가 명시한 결정**을 날짜와 함께 기록한다. 목적: 다음 세션·다른 모델이 "개선"이랍시고 뒤집는 것을 방지.
> AI는 기존 결정을 변경하기 전에 반드시 이 로그를 확인하고, 변경은 **사용자 재승인**이 있을 때만.
> 최상위 결정 몇 개는 CLAUDE.md §5에도 요약해 둔다 (그 절이 늘 컨텍스트에 있으므로).

## 기록 형식

```
### {{결정 제목}} ({{YYYY-MM-DD}}, 사용자 명시)
- 결정: {{무엇을 하기로/하지 않기로 했는가 — 구체적으로}}
- 이유: {{왜 — 이걸 적어야 나중에 "이유가 사라졌으니 바꿔도 되나"를 판단할 수 있다}}
- 영향 범위: {{이 결정이 묶는 파일·기능}}
- 폐기 조건: {{있다면 — "○○이 되면 재검토"}}
```

## 결정 목록

### {{예시: 저장 키 변경 금지}} ({{날짜}}, 사용자 명시)
- 결정: localStorage 키 '{{키}}'와 상태 persist 키 '{{키}}'는 변경 금지.
- 이유: 기존 사용자 저장 데이터 호환.
- 영향 범위: {{store 파일}}.
- 폐기 조건: 마이그레이션 코드를 갖춘 메이저 업데이트.

### {{예시: 보상 경제}} ({{날짜}}, 사용자 명시)
- 결정: {{희귀 보상}}의 드롭 경로는 {{열거}}뿐. 재획득 경로 추가 금지.
- 이유: 희소성이 동기 설계의 핵심.
- 영향 범위: {{보상 관련 파일}}.
