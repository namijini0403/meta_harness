# 합리화 방지표 (실패 기록 축적 장치)

> **하네스는 실패를 먹고 자란다.** 실패가 날 때마다 여기에 한 줄 추가하는 것이 하네스 유지보수의 전부다 (HARNESS.md §9).
> 형식: 그때 든 유혹(합리화) → 실제로 벌어진 일 → 규율.
> 규율이 반복 적용되면 CLAUDE.md 불변식으로 승격, 기계 판정 가능하면 테스트로 전환.

## 범용 시드 (실전에서 검증된 것 — 프로젝트에 맞게 취사선택)

| 유혹(합리화) | 실제로 벌어진 일 | 규율 |
|---|---|---|
| "부분 테스트만 돌려서 통과했으니 됐다" | 전체 실행에서만 잡히는 교차 파일 위반 42건 잠복 | 교차 검사류는 **전체 실행**(필터 금지). |
| "전체 테스트 돌리다 남의 파일 위반이 보이니 같이 고치자" | 병렬 형제 에이전트와 동시편집 충돌 | **내 경계 안 파일만** 수정, 타 파일 위반은 보고만. |
| "이 범위는 대충 이 정도 개수지" | 과소집계 → 커버리지 누락 (프로브가 적발) | 개수는 정본 문서·grep으로 정확히 집계. 추정 금지. |
| "이 산출물은 감사에 안 걸리니 대충" | 감사 밖 영역에 버그 202건 잠복 (후에 감사 편입돼 드러남) | 감사가 안 보는 곳일수록 육안 필수. "감사 있으니 대충" 금지. |
| "자동 검증기가 통과시켰으니 동작한다" | 표시값과 정답이 따로 노는 실버그 — 자동 테스트는 정답만 보고 표시를 안 봄 | 독립 재계산(B) + 육안(C)는 생략 금지 (HARNESS.md §6). |
| "통제 어휘에 없는 토큰이니 그냥 추가하자" | 무단 확장으로 다중 동기화 지점 붕괴 | 통제 어휘 확장은 제안 → **메인 승인** 후 등록. |
| "임시 파일 이름은 아무거나" | 자동정리 훅이 산출물을 삭제 | 임시 파일 접두사 규약 준수, 제출 전 전부 삭제. |
| "이 결정은 이전 세션 거니까 더 나은 방식으로 바꾸자" | 사용자 명시 결정을 뒤집어 재작업 | 불변 결정 로그 확인 전 기존 결정 변경 금지. |
| "테스트 수가 줄었지만 내 작업과 무관할 것" | 회귀를 커밋 | 기준선 미달 시 원인 규명 전 커밋 금지. |
| "git add -A로 한 번에" | 커밋 금지 파일(저작권 원본·시크릿)이 이력에 영구 잔존 | 명시 경로만 add. |
